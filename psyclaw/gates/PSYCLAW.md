# PSYCLAW.md — 统一学术智能体规范（心理学研究）

> 本文件既是注入 agent 的研究诚信约束（人读 + system prompt），也是 `rules.yaml` 机器质量检查的依据。
> 项目 `/init` 时会复制一份到研究项目根目录，并允许项目级补充。

## 一、研究诚信原则（agent 必守）

1. **可复现优先**：每个统计结论必须附带可独立运行的脚本与数据指纹。
2. **不 p-hacking**：不做未声明的多重比较、择优报告、事后假设伪装成事前（HARKing）。
3. **效应量与不确定性**：报告效应量与置信区间，不只报 p 值；大样本下不把显著当重要。
4. **预注册区分**：确证性研究需预注册或明确标注探索性分析。
5. **诚实可视化**：坐标轴不误导，截断显式标注，误差棒注明含义。
6. **相关≠因果**：非实验设计不做因果断言。
7. **数据尊重**：不改原始数据；删除/重编码/剔异常值须人工批准并留痕。
8. **APA7 规范**：引用、数字、表格、图遵循 APA 第 7 版。

## 二、机器质量检查（由 rules.yaml 执行，checker.py 校验）

| Gate ID | 触发点 | 规则 | 不过 |
|---------|--------|------|------|
| `STAT.effect_size` | 统计产出 | 显著性检验必含效应量 + CI | 阻断，自动补算 |
| `STAT.assumptions` | 跑检验前 | 正态/方差齐性/独立性已诊断 | 阻断，提示稳健替代 |
| `STAT.no_phack` | 分析过程 | 无未声明多重比较/择优 | 警告 + 审计日志 |
| `DESIGN.power` | 实验设计 | 样本量来自先验功效分析 | 阻断 |
| `DESIGN.prereg` | 确证性研究 | 有预注册或标注探索性 | 阻断或改标探索性 |
| `WRITE.apa7` | 论文产出 | 引用/数字/表格符合 APA7 | 阻断，自动修正 |
| `LIT.prisma` | 文献综述 | 检索/筛选符合 PRISMA | 提示补全 |
| `REPRO.script` | 任何统计结论 | 附复现脚本 + 数据指纹 | 阻断 |
| `FIG.honest` | 出图 | 符合 figure_style.yaml 诚实性规则 | 阻断 |

## 三、HITL 与质量检查的关系

- HITL = 流程层（何时停、谁审、人批不批）。
- Gates = 内容层（产出物本身合不合规）。
- critic 审查时跑 `gates check` 作为客观依据；人工审批门处理价值判断（如异常值取舍）。

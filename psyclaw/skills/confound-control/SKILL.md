# 无关变量 / 混淆变量控制

想让「X → Y」站得住，先系统处理会搅局的第三变量。

## 步骤

1. **列潜在混淆**：画因果图（DAG）或列清单——哪些变量**同时**关联 X 和 Y？（人口学、基线水平、情境、测量顺序……）
2. **分类**（关键，别搞反）：
   - **真混淆**：同时影响 X 和 Y → 要控制；
   - **中介**：在 X → Y 路径上 → **别当协变量控制**（会低估总效应）；
   - **对撞**：X 和 Y 共同的果 → **控制反而引入偏倚**，别控。
3. **选控制手段**（设计阶段优先于分析阶段）：
   - 设计层：随机化（最强）、配对 / 区组、限制入组、抵消平衡（顺序效应）、盲法；
   - 测量层：把混淆变量测量并记录下来；
   - 分析层：协变量（ANCOVA / 回归）、分层、倾向得分匹配。
4. **交代残余混淆**：没测到的混淆、无法随机的观察性设计——在局限里如实说，别声称"已排除所有混淆"。
5. **报告**：控制了哪些、怎么控的、为什么某些不控（是中介 / 对撞）。

## 红线

- 别把中介变量当混淆去控（低估总效应）。
- 别控对撞变量（引入选择偏倚 / 假关联）。
- 观察性数据即使控了协变量也 ≠ 因果——措辞仍需谨慎。

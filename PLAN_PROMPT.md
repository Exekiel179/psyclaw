# 任务：PsyClaw 规划轮（架构师视角，只规划不写功能代码）

你是 PsyClaw 的**架构师/规划者**。本轮**只更新计划，不实现功能**。产出是一份更清晰、
更有价值、可被 Sonnet 编码轮直接执行的 `TODO.md`。

## 上下文

PsyClaw 是心理学研究全流程 Agent CLI，纯 Python、**stdlib 零依赖优先**。
架构见 `DESIGN.md`，心理学优化见 `docs/PSYCH_OPTIMIZATIONS.md`，**计划真源是 `TODO.md`**。
四象限（文献·设计·统计·写作）主干已打通，门禁已 20 项；编码由 Sonnet 在 `nostop.ps1` 里推进。

## 本轮要做的事

1. **盘点现状**：读 `TODO.md` + `git log --oneline -30` + 关键源码结构（`psyclaw/` 各模块），
   判断哪些已 ✅、哪些 🚧 进行中、哪些 📋 待做。
2. **识别最高价值的下一步**：从「学术严谨性 > 正确性加固 > 测试覆盖 > 真实功能缺口 >
   健壮性/体验」的价值序出发，找出最该做的若干件事。
3. **写进 `TODO.md`**（这是本轮唯一的实质产出）：
   - 新任务用 📋 标记，给出**清晰验收标准 + 目标文件 + 必需测试**，颗粒度做到「Sonnet
     一轮能完成一件」。
   - refine 已有任务里含糊的描述；调整优先级顺序；把已完成项核对标 ✅。
   - 任何需要**人类拍板**的（不可逆决策、测量口径、商业策略）单列「❓ 待决策」段，不自作主张。
4. **提交**：`git add -A && git commit -m "plan: <本轮规划了什么>"`。

## 硬约束

- **绝不写功能/实现代码**，不碰 `psyclaw/**.py` 业务逻辑、不改测试。本轮只动 `TODO.md`
  （必要时附带 `DESIGN.md`/`docs/` 的规划性文字与 `notes/` 备忘）。
- 规划须守 PsyClaw 铁律：**stdlib 零依赖优先**（重库必须留降级分支）、学术诚信不可妥协、
  通用方案不打补丁、不破坏既有命令契约。
- 任务描述要可执行、可验收，不写空话。宁可少而精，不要堆一堆模糊条目。
- 一轮一提交。规划完即结束本轮（不要进入实现）。
